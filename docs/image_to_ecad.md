# 이미지 → 도면 (Phase 5 워크플로)

손그림/순서도/도면 **이미지**를 편집가능한 Easy CAD 도형으로 복원하는 초안 워크플로.
"완벽 복원"이 아니라 **"고품질 초안 → 사용자 수동보정"**.

## 어디서 일어나나

앱 안이 아니라 **Claude Code 세션 안**에서 일어난다. 런타임 AI·외부 게이트웨이 의존이 0이라
(전북대 gateway 쿼터와 무관) 지속성이 높다. 산출물 `.ecad`를 사용자가 앱에서 평소처럼 연다.

```
이미지 → [Claude vision, 앞단] → [sketch_build 빌더, 뒷단] → out.ecad → 앱에서 Ctrl+O
```

- **앞단(이미지 이해):** Claude가 `Read`로 이미지를 직접 보고 상자·심볼·화살표·라벨·대략적 위치를 읽는다.
- **뒷단(도형 생성):** `easycad/fileio/sketch_build.py`의 `Sketch` 빌더(순수 파이썬, Qt 비의존)를
  호출해 `.ecad` JSON을 만든다. 바인딩·변환원점 같은 함정을 빌더가 감춰 항상 유효한 파일이 나온다.
- **좌표:** 이미지 픽셀을 그대로 캔버스 좌표로 쓴다. 사용자가 열어서 스케일·정리한다.

## Claude가 하는 일 (절차)

1. 사용자가 준 이미지를 `Read`로 본다.
2. 상자/심볼과 그 대략적 위치(x·y·w·h, 이미지 픽셀)·라벨, 화살표 연결(누구→누구)을 파악한다.
   ⚠ **엣지는 그려진 것만 옮긴다** — "이 흐름이면 여기로 돌아가야 논리적"이라며 원본에 없는 화살표를
   추가하지 말 것(실조건 함정 2026-07-21: `오류메시지→로그인` 반환선을 없는데 만들어 여러 번 재작업).
   dead-end로 보이면 그대로 dead-end. 각 화살표를 원본과 1:1 대조한다.
3. `Sketch` 빌더를 호출하는 짧은 파이썬 스크립트를 짜서 실행 → `out.ecad` 생성.
4. 사용자에게 "앱에서 `Ctrl+O`로 `out.ecad`를 열어 확인·보정하세요"라고 안내.

## 빌더 API

```python
from easycad.fileio.sketch_build import Sketch

s = Sketch()
# 도형 — 좌표·크기는 이미지 픽셀. label이면 도형 중앙에 텍스트.
a = s.symbol("terminal", 200, 60, 200, 80, "시작")   # 순서도 심볼(6종)
b = s.symbol("decision", 230, 220, 150, 120, "조건?")
c = s.box(520, 245, 200, 80, "처리")                 # 직사각형
d = s.ellipse(230, 420, 150, 90, "끝")               # 타원/원
# 화살표 — 두 도형을 잇는다. 양끝이 도형 변에 지속연결되고 직교 엘보로 자동 라우팅된다.
s.arrow(a, b)
s.arrow(b, c, label="예")                            # 화살표에 라벨(중점 위쪽)
s.arrow(b, d, label="아니오")
s.text(200, 20, "제목")                              # 도형에 안 붙는 자유 텍스트
s.save("out.ecad")
```

### ⚠ 연결 도형은 중심축을 맞춘다 (화살표가 반듯해지는 핵심)

화살표는 두 도형의 **변 중점 포트**를 잇고 직교 라우터가 엘보를 만든다. 두 포트의 축이 몇 px만
어긋나도 라우터가 그 차이를 **작은 계단(꺾임)**으로 처리한다. 그래서 이미지를 읽을 때 노드를
**격자에 정렬**해야 한다(Mermaid import가 반듯한 이유 = BFS 격자 배치):

- **세로로 잇는 도형**들은 **같은 중심 x**(`x + w/2`)에 놓는다 → 세로 화살표가 반듯.
- **가로로 잇는 도형**들은 **같은 중심 y**(`y + h/2`)에 놓는다 → 가로 화살표가 반듯.
- 마름모(decision)·원(ellipse)은 꼭짓점/극점이 **중심축에만** 있으므로, 이들과 잇는 상대 도형을
  그 중심축에 맞추는 게 특히 중요하다.

예: `symbol("terminal", 300, 60, 200, 80)`(cx=400) 아래에 `symbol("decision", 320, 190, 160, 120)`
(cx=400)를 두면 둘 다 cx=400 → 세로 화살표 반듯. 손그림이 삐뚤어도 초안은 격자로 정렬해 뽑는다
(사용자가 원본 배치를 원하면 이후 옮긴다).

### ⚠ 밀집 순서도는 포트 방향을 지정한다 (`from_side`/`to_side`)

화살표는 기본적으로 상대 도형을 향하는 **최근접 변 중점**에 붙는다. 단순 도표는 이걸로 충분하지만,
밀집 순서도(루프·다갈래 분기·합류)에서는 여러 화살표가 같은 변에 몰려 겹친다. `arrow(...,
from_side=, to_side=)`(값 `"N"·"E"·"S"·"W"`)로 접속 변을 못박아 분산한다:

- **정렬된 두 노드 사이 피드백 루프**(예: `저장?`⇄`정보입력`, 아래→위): 양끝을 **같은 측면**으로 빼
  본선과 겹치지 않게. `s.arrow(save, inp, from_side="E", to_side="E", label="아니오")` → 오른쪽 측면 루프.
- **다갈래 분기**(한 노드→여러 노드): 부모는 `from_side="S"`, 각 자식은 `to_side="N"` → 하단 버스처럼.
- **합류**(여러 노드→한 노드): 각 노드 `to_side="N"`(또는 좌우 분산) → 대상 상단으로 병합.
- **큰 루프백**(맨 아래→맨 위): `from_side="E", to_side="E"` → 먼 오른쪽으로 돌아 올라감.

지정 안 하면 최근접(기존 동작) — 단순한 흐름엔 그대로 두는 게 낫다. 한 다이어그램 안에서 완성도는
밀집도에 반비례하니, 겹치는 곳만 골라 힌트를 준다(전부 지정하면 오히려 과함).

**긴 루프백은 `channel_x`/`channel_y`로 외곽 우회.** 코어 라우터는 도형은 완벽히 피하지만 **다른
화살표(선)는 장애물로 안 본다**(되먹임 위험 때문). 그래서 맨 아래→맨 위 같은 **긴 루프백**은 최단
경로를 찾다 내부를 가로질러 다른 화살표와 겹친다. 이런 화살표만 외곽 채널로 손수 뺀다:

```python
# 로그아웃→로그인 큰 루프백: 모든 열보다 오른쪽(x=1180) 세로 채널로 돌아 올라감
s.arrow(logout, login, from_side="E", to_side="E", channel_x=1180, label="아니오")
```

`channel_x=X`면 `[출발포트 → (X,출발y) → (X,도착y) → 도착포트]` 4점 경로로 x=X 세로 채널을 탄다
(`channel_y`는 가로 채널). X는 **가로질러선 안 되는 모든 도형/열보다 바깥** 값으로 준다. 채널을 주면
그 화살표는 auto_route=False(도형 이동 시 끝점만 추종, 채널 경로는 고정). 짧은 화살표엔 쓰지 않는다
— 내부를 가로지르지 않으면 채널이 오히려 돌아가 보인다.

### ⚠ 밀집 순서도 실전 지침 (2026-07-21 실조건서 확인)

로그인 순서도(루프·4갈래 분기·합류) 실조건 검증서 얻은 규칙. **계단은 코어(Stage4 근접정렬 흡수)가,
겹침은 아래 빌드 힌트가 잡는다** — 둘은 역할이 갈린다.

- **열별 박스 폭 통일** — 세로로 잇는 도형은 같은 cx뿐 아니라 **같은 폭**으로. 폭이 다르면 E/W 포트
  x가 어긋나 피드백 루프에 작은 계단이 남는다(Stage4가 못 잡는 케이스 — 부착점이 도형 밖이라). 가로로
  잇는 도형은 **같은 중심 y**(예: `로그인성공?`↔`오류메시지`).
- **피드백 루프(아래→위, 같은 열)** — `from_side/to_side` 같은 측면 + `channel_x`(열 변 밖 ~20px)로
  깔끔한 U-bump. 예: `s.arrow(save, inp, from_side="E", to_side="E", channel_x=열cx+폭/2+20)`.
- **여러 화살표 합류(N개→1)** — 각 화살표에 **같은 `channel_y`**(대상 위 공통 레일)를 줘 **병합 버스**로
  모은다. 안 주면 각자 다른 높이로 대상 한 점에 몰려 겹친다. 예: 4열 하단→`로그아웃?` 전부 `channel_y=레일`.
- **여러 반환이 같은 도형으로** — 원본이 하나의 긴 선(트렁크)으로 합치면 **같은 `channel_x`**를 줘
  **병합**한다(collinear라 한 선으로 보임 = 합류 트렁크, 원본과 일치). 예: `오류메시지→로그인`과
  `로그아웃→로그인`을 둘 다 `channel_x=1050`. 반대로 **서로 다른 목적지**로 가는 루프백은 `channel_x`를
  다르게 줘 분리한다. ⚠ 같은 대상인데 값만 살짝 다르게 주면 두 선이 나란히 겹쳐 지저분하다(실조건 함정).

### 심볼·색

- **심볼 kind:** `decision`(마름모)·`terminal`(스타디움)·`data`(평행사변형)·`prep`(육각형)·
  `document`(문서)·`database`(원기둥).
- **색·두께:** 선택 인자 `color=`(기본 검정 `#000000`)·`width=`(기본 6)·`fill=`(기본 없음).
  색은 `#RRGGBB`·`#AARRGGBB`(alpha 먼저)·`#RGB` 허용.
- 반환된 `Node`를 `arrow(src, dst)`에 넘기면 빌더가 서로 마주보는 변 중점 포트를 골라 잇는다.

## 완성도 기대치 (입력별)

| 입력 | 기대 |
|---|---|
| 깔끔한 순서도 이미지 | 도형·연결 대부분 정확 |
| 손그림/화이트보드 사진 | 배치·연결 근사 초안(수동보정 전제) |
| 임의 도면/스케치 | best-effort, 지원 도형 어휘 내 |

손글씨 텍스트는 Claude가 오독할 수 있다(수동보정 전제).

## 스코프 밖 (승인됨)

- 앱 내장 버튼/단축키(런타임 AI) — 후속 승격(A) 여지만 열어둠.
- Mermaid 경유 자동배치 — 원본 위치를 버리므로 배제(별도 '정돈 모드'로 보완 여지).
- DXF 대상 아님(`.ecad`·PDF는 기존 아이템 직렬화 재사용).
