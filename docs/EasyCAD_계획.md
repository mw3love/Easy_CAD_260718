## 3. 아키텍처 핵심 결정

**A. 좌표계 / 단위**
CAD 왕복과 A-size 인쇄를 위해 **mm 단위 월드좌표**를 채택(화면 픽셀과 분리). 무한 캔버스는 월드좌표 위에서 팬/줌. Y축 방향(CAD는 Y-up, 화면은 Y-down)은 DXF 입출력 시 변환.

**B. 문서 모델 (가장 중요한 리팩터 지점)**
pasteflow 편집기는 QGraphics 아이템을 사실상 모델로 쓰는 구조로 보임(`확인 필요` — grep만 했고 완독 안 함). Easy CAD는 결국 **모델↔뷰 분리**가 필요하다:
- 이유: DXF 매핑 · Undo/Redo · 저장/열기 · "객체 개별 인식" 전부 깨끗한 엔티티 모델을 전제로 함.
- 방침: v1은 속도를 위해 기존 아이템 구조를 최대한 재사용하되, **각 아이템이 직렬화 가능한 의미정보(타입·기하 파라미터)를 반드시 지니게** 설계한다. 정식 문서모델은 Phase 3(DXF 직전)에 도입 — 그때 리팩터 규모는 pasteflow 코어 완독 후 확정.

**C. Undo/Redo**
`QUndoStack` 기반 커맨드 패턴으로 Phase 1부터 도입(뒤에 붙이면 고통). 모든 편집을 커맨드로.

## 5. 제안 추가 기능 (사용자 요청으로 검토 — 이 도메인에 특히 잘 맞는 것)

참고 도면(방송 결선도·CCTV 망·랙 배치)을 분석해 **실제 사용처에 killer가 될 기능**:

1. **심볼/스텐실 라이브러리** ★ — 참고 도면은 증폭기·카메라·랙·안테나 같은 블록을 반복 사용. Visio 스텐실처럼 재사용 심볼 팔레트를 두면 작도 속도가 급상승. (이 도메인 최고 가치)
2. **직교 커넥터 + 자동 라우팅 + 점프오버** ★ — AutoCAD 결선도는 대부분 직교선. 베지어(순서도)와 **직교선(결선도)** 둘 다 지원해야 함. 선 겹칠 때 점프오버.
3. **포트/커넥션 포인트** — 도형에 접속점을 정의(참고 도면의 포트 마커처럼). 스냅이 포트에 착 붙음.
4. **정렬/분배**(align/distribute) — 깔끔한 도면의 핵심, lucid에 있음.
5. **그리드 + 눈금**, **미니맵**(무한캔버스 큰 도면 탐색), **스타일 복사**(format painter).
6. (advanced) **케이블/넷 번호 자동채번** — 참고 도면에 케이블번호가 도처에 있음. 도메인 특화 자동화.

→ 우선순위: **1·2·3을 Phase 2~3에 흡수** 권장(결선도 실사용에 직결). 4·5는 Phase 4, 6은 후보.
